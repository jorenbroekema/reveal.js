Javascript ES6 & lit-html rendering

What is ES6?

Block scoping, let and const

var

Vars get hoisted to lexical scope before anything else happens.

              
              function fn () {
                if (true) {
                  var y = 1 // hoisted up to function scope
                }
                console.log(y); // 1
              }

Let

              
              function fn () {
                if (true) {
                  let y = 1 // only inside this if
                  console.log(y) // 1
                }
                console.log(y) // undefined
              }

const

              
              function fn () {
                if (true) {
                  const y = 1 // only inside this if
                  console.log(y) // 1
                }
                console.log(y) // undefined
              }

Const works exactly like let, but cannot be reassigned!

Always use let and const, NEVER var

ES Modules

The new standard for importing and exporting packages

Exports

Before: 

                // Default export
                module.exports.default = function () { return 'Hello, World' };

                // Named export
                module.exports.pi = 3.14159;
After: 

                // Default export
                export default function () { return 'Hello, World' };

                // Named export
                export const pi = 3.14159;

Imports

Before: 

              // Default import
              const arbitraryName = require('./library.js').default ||
                require('./library.js');

              // Named import
              import { libraryClass } from './library.js';

              // Import all named Imports
              require('./helpers');

              // Import as custom named
              const Helpers = require('./helpers');

Imports

After: 

              // Default import
              import arbitraryName from './library.js';

              // Named import
              import { libraryClass } from './library.js';

              // Import all named Imports
              import './helpers';

              // Import as custom named
              import * as Helpers = import('./helpers');

Script type module


              <script type="module" src="./helpers.js"></script>

Bare module resolution


              // relative import
              import { html, render } from './node_modules/lit-html/index.js';

              // bare import
              import { html, render } from 'lit-html';

Bare module resolution

For bare imports, you need something that resolves them:

  • Build tools like Webpack, Grunt, Rollup
  • Local Dev Servers like:
    • es-dev-server
    • webpack-dev-server
    • polymer-cli

Template literals


              const animal = 'squirrel';
              const food = 'apple';

              // Before
              console.log('The ' + animal + ' eats the ' + food);

              // With template literals ` `
              console.log(`The ${animal} eats the ${food}`);

              // Or in multiline strings
              console.log(`
                Animal: ${animal},
                Food: ${food}
              `);

What is lit-html?

Preact rendering (VDOM)

Lit-html rendering

How do we use it?

Template Literals!


              import { html, render } from 'lit-html';

              // A lit-html template uses the `html` template tag:
              let sayHello = (name) => html`<h1>Hello ${name}</h1>`;

              // It's rendered with the `render()` function:
              render(sayHello('World'), document.body);

              // And re-renders only update the data that changed
              render(sayHello('Everyone'), document.body);

How it works

Only the dynamic parts of the DOM are updated.

Only when the property value has changed.

Advanced usage


              import { html, render } from 'lit-html';

              // A lit-html template looping over an array to generate a list
              const listAnimals = (animals) => html`
                Animals:
                <ul>
                ${animals.map(animal => html`
                  <li>
                    ${animal}
                  </li>
                `)}
                </ul>
              `;
              render(listAnimals(['Giraffe', 'Monkey', 'Dog']), document.body);

Advanced usage


              import { html, render } from 'lit-html';

              const userAuthorized = true;
              const authorizedMessage = "You're authorized!";
              const unauthorizedMessage = "You don't have access to this.";

              // A lit-html template looping over an array to generate a list
              const listAnimals = (animals) => html`
                <h3>Dashboard</h3>
                ${userAuthorized ?
                  return html`${authorizedMessage}` :
                  return html`${unauthorizedMessage}`
                }
              `;

              render(listAnimals(['Giraffe', 'Monkey', 'Dog']), document.body);
